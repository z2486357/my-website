Box Counter


The question is :
If you buy something online, when you receive your things, sometime you will find that you get a big box to load your little things.
For those shop, probably they just randomly pick a box that can load all your things so there is still a lot of space and this is a kind of waste.
The goal of this application is to calculate the boxex needed of different size and how to put.

Assumption 1 : There are many different kind of boxes available.

x1*y1*z1

x2*y2*z2

x3*y3*z3

x4*y4*z4

x5*y5*z5

Assumption 2 : Every thing will be determine as a minimum cuboid.

Algorithm

1. First you declare the kind of box you have (x1*y1*z1 / x2*y2*z2 / ......)
2. Declare the size of all the goods (xx1*yy1*zz1 / xx2*yy2*zz2 / ......)
3. After typing the number of each goods, it will output the box(es) you need and how to put it.
4. The optimal solution is the fewest and smallest box(es).
